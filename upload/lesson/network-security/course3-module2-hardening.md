# Network Hardening Best Practices

Hardening reduces attack surface before incidents occur.

## Hardening Checklist

- Change default credentials immediately.
- Disable insecure protocols (Telnet, FTP).
- Patch network devices consistently.
- Restrict management interfaces by IP.
- Use VLANs and segmentation.

## Ongoing Activities

- Continuous vulnerability scanning.
- Configuration backups and review.
- Access control audits.

A hardened network is not unbreakable, but it is significantly harder to exploit.
