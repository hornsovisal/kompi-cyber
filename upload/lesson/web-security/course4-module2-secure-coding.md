# Secure Coding Practices

Secure coding is about preventing vulnerabilities during development.

## Principles

- Validate all untrusted input.
- Handle errors safely.
- Avoid exposing sensitive data.
- Use strong authentication and authorization checks.

## Team Practices

- Code review with a security checklist.
- Dependency scanning.
- Static analysis and linting.
- Threat modeling for key features.

Security should be built in, not added later.
